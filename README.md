## 📌 Tentang SiMudah
Sektor usaha dagang mikro seperti warung sembako dan pedagang makanan kecil sering menghadapi masalah ketidaksinkronan informasi stok dengan pelanggan, serta kesulitan dalam melacak arus keuangan secara akurat.

**SiMudah** hadir sebagai solusi digital praktis untuk digitalisasi warung dan pedagang mikro. Platform berbasis web ini berfokus pada efisiensi operasional melalui transparansi stok barang, sistem pemesanan terjadwal (*pre-order* terbayar), dan otomatisasi pencatatan laporan keuangan sederhana. Dengan pendekatan *Mobile-First*, aplikasi ini dirancang sangat ringan agar optimal diakses oleh pedagang dengan perangkat spesifikasi rendah sekalipun.

---

## 🚀 Fitur Utama & Unique Selling Point (USP)

### 🌟 Fitur Utama
* **Digital Product Catalog:** Pedagang dapat mengunggah katalog barang/menu dengan status stok yang diperbarui secara mandiri.
* **Paid Pre-Order System (Self-Pickup):** Konsumen memesan dan membayar di awal, lalu mengambil pesanan langsung ke lokasi tanpa perlu mengantre.
* **Real-time Payment Verification:** Integrasi API QRIS untuk memverifikasi status pembayaran otomatis sebelum pesanan masuk ke antrean.
* **Smart Financial Ledger:** Pencatatan otomatis pendapatan kotor harian ditambah fitur input beban biaya operasional (listrik, air, sewa) untuk laporan laba/rugi bersih.

### 🛡️ Unique Selling Point (USP)
* **Zero-Queue Shopping:** Mengeliminasi waktu tunggu di lokasi secara instan dibandingkan pemesanan manual via WhatsApp.
* **Simplified Bookkeeping:** Berfungsi sebagai asisten keuangan instan bagi pedagang kecil tanpa kalkulasi manual yang rumit.
* **Reliability for Small Merchants:** Aplikasi web yang ringan, responsif, dan hemat kuota internet.

---

## 🛠️ Tech Stack (Tumpukan Teknologi)

* **Frontend:** Blade Templating Engine (Laravel Native) & Tailwind CSS *(Utility-first framework untuk antarmuka modern & ringan)*.
* **Backend:** PHP dengan Framework Laravel *(Fondasi aman, manajemen DB efisien, pengembangan cepat)*.
* **Database:** SQLite *(Penyimpanan relasional yang terstruktur dan sangat portabel)*.
* **Third-Party Integration:** Payment Gateway API (QRIS) untuk verifikasi pembayaran *real-time*.

---

## 📐 Batasan Aplikasi (Scope)
* Aplikasi hanya melayani metode **Self-Pickup** (ambil sendiri di warung), tidak menyediakan layanan pengiriman (*delivery*).
* Pengurangan stok barang utama dilakukan secara mandiri oleh pedagang lewat *dashboard* untuk menjaga fleksibilitas inventaris fisik.
* Dioptimalkan penuh untuk perangkat *mobile browser* (Android/iOS) dan mendukung browser desktop terbaru.

## 🗺️ Rencana Pengembangan (Roadmap)

- [x] **Fase 1 (MVP - Minimum Viable Product):** Multi-role login (Pedagang & Pengguna), katalog stok, sistem *pre-order*, dan integrasi QRIS.
- [ ] **Fase 2 (Enhancement):** Laporan laba/rugi bersih (input beban operasional) dan notifikasi status pesanan via WhatsApp/Email.
